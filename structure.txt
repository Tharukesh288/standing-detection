Standing Detection System for Bus                                         
                                                                                
 Context                                                                      
                                                                                
 User wants to build a system that detects standing people inside a bus and
 displays the count on a website. The system should:                            
 - Run on a low-budget edge device (or local computer for testing)
 - Use a pre-trained YOLO model for person detection
 - Use a USB webcam for video capture
 - Distinguish between standing and sitting people
 - Display real-time count on a web interface

 Recommended Approach

 Architecture Overview

 [USB Camera] → [YOLO Detection] → [Standing/Sitting Classifier] → [Flask API]
 → [Web Dashboard]

 Components

 1. Detection Backend (Python + OpenCV + YOLO)

 - File: backend/detector.py
 - Uses YOLOv8n (nano - smallest, fastest, good for edge devices)
 - Captures video from USB webcam
 - Detects people and classifies as standing/sitting based on:
   - Bounding box aspect ratio (standing = tall/narrow, sitting = wide/short)
   - Or pose estimation (keypoint detection for more accuracy)
 - Runs a Flask/FastAPI server that serves:
   - WebSocket for real-time count updates
   - REST API for current count
   - Optional: MJPEG stream of processed video

 2. Web Dashboard (HTML + JavaScript)

 - File: frontend/index.html
 - Simple, lightweight static page
 - Connects via WebSocket for real-time updates
 - Displays:
   - Current standing count (large, easy to read)
   - Optional: Live video feed
   - Optional: Historical chart

 3. Deployment Options

 - Edge Device: Raspberry Pi 4 (or Pi 5 for better performance)
 - Local Computer: For development/testing
 - Cloud Dashboard: Optional - use ngrok or deploy frontend to Vercel/Netlify

 Technical Details

 Standing vs Sitting Detection

 Two approaches (will implement the simpler one first):

 1. Aspect Ratio Method (simpler, faster):
   - Standing: height > 1.5 × width
   - Sitting: height ≤ 1.5 × width
   - Requires camera to be mounted at an angle that captures full body
 2. Pose Estimation (more accurate, heavier):
   - Use YOLO-pose or MediaPipe
   - Check knee/hip angles to determine standing vs sitting

 YOLO Model Selection

 - YOLOv8n (nano): ~3MB, fastest, good for edge devices
 - YOLOv8s (small): ~11MB, better accuracy if device has capacity

 2. Test standing vs sitting classification with sample images
 3. Start Flask server - verify API endpoints work
 4. Open frontend in browser - verify count updates in real-time
